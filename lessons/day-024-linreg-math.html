The simplest ML model — and a great teacher

Linear regression is the "hello world" of ML, but underestimated. It teaches you, in one tidy package, every concept you'll re-meet in deeper models: the model, the loss, the closed-form solution, the iterative (gradient descent) solution, and how to evaluate fit. Internalize this and most ML papers become easier to read.

Today: the math. Tomorrow: the code (NumPy from scratch, both ways).

The model

You have data: n examples, each with d features. X is the n×d data matrix. y is the n-vector of targets. The model assumes there's some weight vector β and a bias β₀ such that:

y_i ≈ β₀ + β₁ x_{i1} + β₂ x_{i2} + ... + β_d x_{id}

Or in matrix form, after augmenting X with a column of 1s for the bias:

y ≈ Xβ

Goal: find β.

The loss

Define "fit quality" by sum of squared residuals — mean squared error (MSE):

L(β) = (1/n) · Σᵢ (y_i − x_iᵀ β)² = (1/n) · ‖y − Xβ‖²

Why squared? Two reasons:

  • It's smooth (differentiable everywhere) so gradient-based methods work.
  • It corresponds to maximum likelihood under Gaussian noise. (If you assume the noise is normally distributed, minimizing MSE is the same as finding the most likely β.)

The closed-form solution — the normal equations

Take the gradient of L w.r.t. β. Set to zero. Solve. You get:

β* = (XᵀX)⁻¹ Xᵀ y

This is the normal equations solution. It exists in closed form because the loss is a convex quadratic in β. No iterations needed — direct formula.

The derivation: L = (1/n)·(y - Xβ)ᵀ(y - Xβ). Expand. Differentiate w.r.t. β: ∇L = (-2/n)·Xᵀ(y - Xβ). Set to 0: XᵀXβ = Xᵀy. If XᵀX is invertible (it usually is for n > d with non-degenerate features), β = (XᵀX)⁻¹Xᵀy.

In practice you don't compute the inverse. You call np.linalg.solve(XᵀX, Xᵀy).

The iterative solution — gradient descent

You can also reach β* by gradient descent. The gradient of MSE is:

∇L(β) = (−2/n) · Xᵀ (y − Xβ)

So the GD update is:

β ← β + (2η/n) · Xᵀ(y − Xβ)

(Or just absorb the constants into η.)

Why two methods?

  • Normal equations are exact but O(d³) (the matrix inverse). Fine for d < 1000-ish.
  • GD is iterative but scales to huge d. Required for neural nets where d is millions.
