From counting to a real theory

Day 6 introduced "(favorable outcomes) / (total outcomes)." That's correct, but it only works for finite spaces where every outcome is equally likely. Today you graduate to the actual mathematical structure of probability: sample spaces, events, and the three axioms that everything else is built on.

Why this matters: when you start dealing with continuous things (a real-valued sensor reading, time-to-failure, model output probabilities), "count favorable outcomes" stops making sense. The axiomatic structure still works.

The three objects

  1. Sample space S — the set of all possible outcomes of an experiment. For one coin: S = {H, T}. For one die: S = {1,2,3,4,5,6}. For "time until website crashes": S = [0, ∞).
  2. Event A — a subset of S. For one die, "roll is even" → A = {2, 4, 6}.
  3. Probability function P — assigns a number in [0, 1] to each event.

Kolmogorov's three axioms

Everything in probability theory is derivable from these three rules. They're the math equivalent of Newton's laws.

  1. P(A) ≥ 0 for every event A. (Probabilities are non-negative.)
  2. P(S) = 1. (Something in the sample space must happen.)
  3. Additivity: if A and B are disjoint (can't both happen), then P(A ∪ B) = P(A) + P(B).

From these three you can derive everything: P(∅) = 0, P(Aᶜ) = 1 − P(A), P(A ∪ B) = P(A) + P(B) − P(A ∩ B) (when not disjoint), and on and on.

Common student trap: "additive" only works when events are disjoint. P(rains tomorrow) + P(sunny tomorrow) ≠ 1 unless you're sure those are the only possibilities AND they can't co-occur. Otherwise use the general formula P(A ∪ B) = P(A) + P(B) − P(A ∩ B).
