From math to code

Yesterday: the math. Today: the code. We'll implement linear regression in NumPy two ways — closed form and gradient descent — and check they give the same answer.

Setup — generate synthetic data

import numpy as np

rng = np.random.default_rng(42)
n = 100
d = 3  # 3 features
X = rng.standard_normal((n, d))
true_beta = np.array([1.5, -2.0, 0.7])
true_bias = 0.5
noise = rng.standard_normal(n) * 0.3
y = X @ true_beta + true_bias + noise

Standard practice: augment X with a column of ones to absorb the bias:

X_aug = np.hstack([np.ones((n, 1)), X])  # shape (n, d+1)
# Now β includes bias as β[0]

Method 1 — Closed form (normal equations)

def fit_closed_form(X, y):
    # X is (n, d), y is (n,). Returns β of shape (d,).
    return np.linalg.solve(X.T @ X, X.T @ y)

beta_cf = fit_closed_form(X_aug, y)
print("closed form β:", beta_cf)
# Expected: ≈ [0.5, 1.5, -2.0, 0.7] (bias, then weights)

One line of math, one line of code (plus the augmentation). Always use np.linalg.solve, never np.linalg.inv.

Method 2 — Gradient descent

def fit_gd(X, y, lr=0.01, n_steps=2000):
    n, d = X.shape
    beta = np.zeros(d)
    losses = []
    for _ in range(n_steps):
        y_pred = X @ beta
        residual = y_pred - y
        loss = (residual ** 2).mean()
        grad = (2 / n) * X.T @ residual   # ← the gradient of MSE
        beta -= lr * grad
        losses.append(loss)
    return beta, losses

beta_gd, losses = fit_gd(X_aug, y)
print("GD β:", beta_gd)
print("closed form β:", beta_cf)
# Should match closely. If not — check lr, n_steps, feature scaling.

Compare them. They should agree to 3-4 decimal places. If they don't, debug:

  • If GD diverges (β explodes): lr too high. Halve it.
  • If GD never converges: lr too low, or features on different scales. Standardize: subtract mean, divide by std for each feature.
  • If they agree but predictions are off: bug in the data or the augmentation.
