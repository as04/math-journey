The day you turn private learning into public signal

You've spent 27 days building skills. Today you turn the linear-regression code from Days 24-25 (plus your Project Euler solutions) into a polished, public GitHub repo. That repo is the start of the portfolio that matters when you apply to Google.

The goal isn't "perfect code." The goal is "code, README, intent, all together, public, with green commit history." A consistent visible output beats sporadic excellent work every time.

The capstone checklist

Work through this top to bottom. ~60 minutes total.

  1. Create a new repo on GitHub: your-username/ml-from-scratch (or a name you prefer).
  2. In the repo, create folders: 
    linear_regression/
project_euler/
README.md
  3. Put your Day 24-25 linear regression code in linear_regression/main.py. Make sure it runs end-to-end with python main.py.
  4. Add a small linear_regression/README.md that explains:
    • What problem this code solves (linear regression).
    • Both methods implemented (closed-form and gradient descent).
    • How to run it.
    • Sample output (paste actual run output).
  5. Put Project Euler solutions (#1, #2, #3 from Days 7 and 14) in project_euler/p001.py, p002.py, p003.py.
  6. Write the top-level README.md. Template below.
  7. Commit and push.
  8. Add the repo link to your LinkedIn / resume.

Top-level README.md template

# ML from Scratch — a learning portfolio

Implementing ML algorithms from first principles in NumPy, no scikit-learn.
This is a learning project — each algorithm is built to be readable,
not optimal.

## Contents
- [`linear_regression/`](linear_regression/) — fit a line, both closed-form
  and via gradient descent. The two methods agree to 4 decimal places.
- [`project_euler/`](project_euler/) — solutions to selected Project Euler
  problems, with both brute-force and math-informed versions.

## Running
```bash
pip install numpy
python linear_regression/main.py
python project_euler/p001.py
```

## About
Built as part of a [self-directed math journey](https://github.com/as04/math-journey).
Updated weekly.

Adjust the link/name. The "About" link is important — it signals you're on a sustained learning arc, not posting one-shot code.

A note on signaling

Recruiters skim repos in 60 seconds. They want to see:

  • README that explains what and why immediately.
  • Code that runs (have a clear "how to run" section).
  • Reasonable commit history (small, descriptive commits — not "wip" × 50).
  • Recent activity (commit graph green within the last month).

You don't need stars. You don't need fancy CI. You don't need contributors. You need evidence of consistent thinking.

Optional: pin your repo

On your GitHub profile page, pin your best 3-6 repos. Put this one near the top. After Month 4 you'll have a neural net repo, an MILP-optimization repo, and a paper-replication repo — all worth pinning.

By Month 6, your GitHub profile is part of your candidacy. By Month 12, it's a primary signal.
