The 2-line answer to "what is a gradient?"

The gradient of a multivariable function f(x, y, ...) is the vector of its partial derivatives:

∇f = (∂f/∂x, ∂f/∂y, ...)

It points in the direction of steepest ascent. Its magnitude is the rate of ascent in that direction.

This is the foundation of every gradient-based optimizer in ML. Tomorrow: gradient descent uses ∇f to know which way to step.

Partial derivatives in 30 seconds

For f(x, y), the partial derivative with respect to x is the derivative of f treating y as a constant. Notation: ∂f/∂x.

Example: f(x, y) = x²y + 3y.

  • ∂f/∂x = 2xy (treat y as constant).
  • ∂f/∂y = x² + 3 (treat x as constant).

So at (x, y) = (2, 1): ∇f = (2·2·1, 2² + 3) = (4, 7). The function increases fastest in the direction (4, 7) from that point.

Why the gradient is perpendicular to level sets

A level set is points where f = const. Moving along a level set doesn't change f, so the directional derivative along that direction is zero. The gradient is the vector that gives the directional derivative as a dot product: ∇f · u. For this to be zero along any direction in the level set, ∇f must be perpendicular to those directions.

This is why every ML optimization trajectory zig-zags across contours but never along them.
