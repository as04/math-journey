The most-asked question in linear algebra

"Solve Ax = b." Every linear regression. Every system of equations. Every step of every numerical solver. The question is: given the matrix A and the vector b, find the vector x that satisfies Ax = b.

Three possible outcomes:

  1. Unique solution. A is invertible (det ≠ 0). x = A⁻¹b. (We almost never literally compute A⁻¹; more on that below.)
  2. No solution. b is not in the column-space of A. The system is inconsistent.
  3. Infinitely many solutions. A has a non-trivial null space; once you find one solution, any vector in the null space added to it is also a solution.

The geometric reframe

Recall (Day 4): A takes input vectors to output vectors via x ↦ Ax. So "solve Ax = b" means: which input x lands at the output b?

  • If A is invertible (det ≠ 0), the transformation is reversible. Every output has a unique input → unique solution.
  • If A is singular (det = 0), it collapses space onto a lower-dimensional subspace (a line or origin in 2D). If b is in that subspace → infinitely many inputs land there. If b is outside → no input lands there.
The columns-of-A view: Ax is a linear combination of the columns of A with weights x₁, x₂, .... So "Ax = b solvable" ⟺ "b is in the span of A's columns."

In practice — don't compute A⁻¹

You will almost never write x = inv(A) @ b in production code. Even though it's mathematically correct, it's:

  • Slower (computing the inverse is more work than solving directly).
  • Less numerically stable (computing inverses amplifies rounding error).

Instead, you call np.linalg.solve(A, b) (NumPy) or solve(A, b) in any library. Internally these use LU decomposition (Gaussian elimination, organized). You get the answer faster and more accurately. Remember this — it's a near-universal interview mark.
