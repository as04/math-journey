Track C: the workhorse of every interview

If sorting is the most-asked algorithm in interviews, hash tables are the most-used data structure. Half of LeetCode problems can be solved in one pass with a dictionary. Every modern programming language ships with one built in (dict, HashMap, etc.) — but if you don't understand how they work, you'll miss when they fail.

The basic idea

  1. You have a key (string, number, anything hashable).
  2. A hash function turns the key into an integer h.
  3. That integer mod table-size gives you an index in an array.
  4. You store the (key, value) pair at that index.
  5. Lookup: hash the key, go to that index, find the value. O(1) on average.

The complication: collisions

Two different keys can hash to the same index. That's a collision. Two main strategies:

  • Separate chaining: each array slot holds a list of (key, value) pairs. Look up the key by scanning the list. Python's dict: open addressing variant; Java's HashMap: chaining.
  • Open addressing: if your slot is taken, try the next one (linear probing) or another deterministic offset (quadratic, double hashing). More cache-friendly, but trickier to delete.

When the table fills up (load factor > ~0.75), it's resized — usually doubled — and all keys re-hashed. This is the amortized "occasionally O(n)" step that keeps average operations O(1).

Worst-case O(n) is rare with good hash functions on real data. But beware: adversarial inputs (someone constructing keys to all collide) can degrade a hash table to a linked list. This is a real security issue called "hash flooding" — modern languages defend with randomized hash seeds.

The interview-pattern unlock

Whenever you see a problem of the form "find pairs / find duplicates / count occurrences / check membership in a stream", your first thought should be hash map. It usually turns an O(n²) brute-force search into O(n) by precomputing a lookup.

Classic example — "two sum" (find two numbers in a list that sum to target):

def two_sum(nums, target):
    seen = {}
    for i, x in enumerate(nums):
        if target - x in seen:
            return seen[target - x], i
        seen[x] = i
    return None

O(n) time, O(n) space. Brute force is O(n²). The hash map is the entire trick.
