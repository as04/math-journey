From outcomes to numbers

So far probabilities have lived in "discrete outcome space" — heads vs tails, die showing 4. But real life is numbers: revenue per user, time-to-click, model loss, profit on a bet. A random variable (RV) is the bridge.

Formally: an RV is a function from the sample space to a real number. For a die: X = the number shown. For two dice: S = sum of dice. For a coin tossed 5 times: H = number of heads.

You then talk about probabilities like P(X = 4), P(X ≥ 3), or summary numbers like the average value E[X].

Expectation: the long-run average

The expected value E[X] is the average value of X if you ran the experiment infinitely many times. For a discrete RV:

E[X] = Σ x · P(X = x)

Worked: roll one die. E[X] = 1·(1/6) + 2·(1/6) + ... + 6·(1/6) = (1+2+3+4+5+6)/6 = 3.5. The die never shows 3.5 — but its average over many rolls converges there.

Linearity of expectation — the killer property

E[X + Y] = E[X] + E[Y]

This holds even when X and Y are not independent. No "ifs," no "buts," no "if they're uncorrelated." Always. This is why expectation is so beloved.

It also extends to multiplication by constants: E[a·X + b] = a·E[X] + b.

The classic example: roll 100 dice. What's the expected sum? Naïve approach: enumerate the 6¹⁰⁰ outcomes. Linearity-of-expectation approach: each die has E = 3.5, sum has E = 100 · 3.5 = 350. Two seconds. This trick collapses entire classes of competitive-programming problems.
