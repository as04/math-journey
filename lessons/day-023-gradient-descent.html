The single algorithm that powers all of ML

Yesterday: the gradient ∇f points in the direction of steepest ascent. Today: if we want to minimize f, we move in the opposite direction. Repeatedly. That's gradient descent.

x_{t+1} = x_t − η · ∇f(x_t)

where η ("eta") is the learning rate — how big a step you take. Almost every deep learning paper ever written uses some variant of this.

Three things go wrong

  1. Learning rate too small: you crawl. Converges, but takes forever.
  2. Learning rate too large: you overshoot and bounce around — or diverge entirely.
  3. Stuck in local minima or saddle points: for non-convex functions (which is every interesting ML loss), GD can get trapped. Variants like momentum, Adam, etc. help.

The modern variants (preview)

  • Stochastic gradient descent (SGD): use a random subset of data for each step. Faster per step, noisier.
  • Momentum: add a fraction of the previous step. Smooths out the trajectory.
  • Adam: SGD + momentum + per-parameter learning rate. The default optimizer for most deep learning. You'll meet it in Month 3.

Implementation, in 8 lines of Python

def gradient_descent(grad_fn, start, lr, n_steps):
    x = start[:]
    for _ in range(n_steps):
        g = grad_fn(x)
        for i in range(len(x)):
            x[i] -= lr * g[i]
    return x

# usage on f(x, y) = x² + y²:
grad_fn = lambda x: [2*x[0], 2*x[1]]
print(gradient_descent(grad_fn, [3, 4], 0.1, 100))  # ~[0, 0]
